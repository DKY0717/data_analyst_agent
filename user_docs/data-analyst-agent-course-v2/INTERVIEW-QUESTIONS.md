# Data Analyst Agent 面试问题

> 本文件用于检验理解，不提供需要逐字背诵的唯一答案。回答时应包含当前项目的真实代码边界、测试证据和未验证假设。

## 1. 项目与架构

1. 为什么这个项目不只是一个 NL2SQL Prompt？
2. 哪些决策由 LLM 完成，哪些必须由确定性代码完成？
3. 为什么选择 LangGraph，而不是一个很长的 Python 函数？

## 2. 安全与权限

1. Intent Guard、SQL Guard 和 Permission Guard 分别防什么？
2. 为什么 SQL Repair 之后必须重新经过 Guard 和权限？
3. 行级过滤与在 Prompt 中要求模型加 WHERE 有什么区别？

## 3. 可靠性与可观测性

1. 网络重试、SQL Repair 重试和用户重试为什么要分开？
2. SQL 已经执行成功但答案生成失败时，API 应该返回什么？
3. LLM 日志为什么不能保存完整 Prompt 和供应商响应？

## 4. 测试与评测

1. SQL 执行成功为什么不能证明业务结果正确？
2. Fake LLM 核心路径能证明什么，不能证明什么？
3. 严格汇总为什么要校验 HEAD、provider、model 和 case 哈希？

## 5. 真实事故

1. MiMo run `29634864907` 为什么失败？
2. 为什么这次失败不能简单解释为“项目 NL2SQL 完全不可用”？
3. 如果继续优化真实评测，你会优先修改超时、重试、分片还是质量阈值？
4. 为什么最终 job 失败，却不能说是质量阈值未达标？

## 6. 项目边界

1. 当前项目哪些能力经过确定性验证，哪些仍依赖外部条件？
2. “具备生产意识”和“经过企业生产验证”有什么区别？
3. 如果接入大型企业 Schema、多租户和高并发，首先需要补什么？

## 7. AI 辅助与个人贡献

1. 这个项目使用 AI 辅助开发，你个人负责了哪些判断与验收？
2. 你如何证明自己不是只能运行 AI 生成的代码？
3. 如果关掉当前仓库，你能从零重建哪些核心能力，哪些还不能？
