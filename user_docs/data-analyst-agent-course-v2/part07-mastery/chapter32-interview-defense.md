# 第32章 面试演示与项目答辩
> 本章预计 1～2 小时，把技术事实组织成可验证、不过度承诺的项目表达。
## 32.1 学习目标
> 能完成 3 分钟介绍、10 分钟演示、架构追问、故障复盘和个人贡献说明。
## 32.2 前置知识
> 已完成课程练习，并能独立运行、调试和重建核心链路。
## 32.3 为什么需要这一模块
> 面试官关注的不只是功能数量，而是需求判断、架构边界、安全意识、验证证据和你是否真正理解代码。
## 32.4 输入、输出与依赖
> 输入是当前代码、测试报告、真实评测事实和个人练习；输出是可现场证明的项目故事与演示预案。
## 32.5 执行流程
```text
problem → architecture → hard decisions → evidence → incident → next improvement
```
## 32.6 当前代码地图
| 内容 | 路径 |
|---|---|
| 演示预检 | `scripts/interview_demo_preflight.py` |
| 证据生成 | `scripts/interview_evidence.py` |
| 学习检查 | `user_docs/data-analyst-agent-course-v2/STUDY-CHECKLIST.md` |
| 问题库 | `user_docs/data-analyst-agent-course-v2/INTERVIEW-QUESTIONS.md` |
## 32.7 关键代码理解
> 答辩只陈述能被代码、测试或报告支持的事实。MiMo 运行可说明 Repair 与结果正确性分片完成、NL2SQL 证据不完整及门禁失败，不能包装成全量通过。
## 32.8 最小动手运行
```bash
python scripts/interview_demo_preflight.py
python scripts/interview_evidence.py
```
## 32.9 故障注入实验
> 演示前关闭模型网络或提供无效配置，练习切换到架构讲解、离线测试和既有脱敏证据，而不是现场失控。
## 32.10 调试路径与常见误判
> 预先准备数据库、端口、配置、演示问题和降级材料；“现场调用一次成功”不等于系统质量证明。
## 32.11 独立编码练习
> 录制一遍 10 分钟答辩：3 分钟背景架构、3 分钟运行、2 分钟安全评测、2 分钟事故与改进，然后删掉所有无法举证的话。
## 32.12 测试或评测验证
> 演示前运行预检、课程契约测试、关键后端测试和前端构建，并记录提交 SHA 与时间。
## 32.13 面试复述题
> 你在这个项目中最重要的三个工程判断是什么，各自有什么证据？
## 32.14 掌握度检查与下一章
> 当你能不看稿画架构、现场定位一次故障、解释门禁失败并重建最小核心时，这门课程完成；之后按变更日志持续更新代码地图。
