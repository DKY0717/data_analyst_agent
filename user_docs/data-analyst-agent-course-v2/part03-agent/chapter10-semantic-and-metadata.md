# 第10章 语义层与元数据目录
> 本章预计 1～2 小时，区分业务概念和物理 Schema。
## 10.1 学习目标
> 能解释销售额等指标为何不能只靠列名猜测。
## 10.2 前置知识
> 需要理解第3章业务表和第9章结构化意图。
## 10.3 为什么需要这一模块
> Schema 说明有哪些字段，语义层说明业务词如何计算，元数据目录负责候选和路由信息。
## 10.4 输入、输出与依赖
> 输入是业务词和 Schema；输出是指标、维度、候选和 JOIN 线索。
## 10.5 执行流程
```text
业务词 → 语义定义/别名 → 元数据候选 → 物理表达式
```
## 10.6 当前代码地图
| 内容 | 路径 |
|---|---|
| 语义 | `backend/app/semantic/` |
| 元数据 | `backend/app/schema_context/` |
## 10.7 关键代码理解
> 阅读 YAML 时关注表达式、默认时间字段、别名和 JOIN；地理维度必须区分客户、订单等查询范围。
## 10.8 最小动手运行
```bash
pytest backend/tests/test_semantic_loader.py backend/tests/test_metadata_catalog.py -q
```
## 10.9 故障注入实验
> 在测试数据中构造同名地理词，观察错误 scope 可能选错字段；不修改正式 YAML。
## 10.10 调试路径与常见误判
> 别名命中只产生候选，不能跳过 Schema 和 JOIN 验证。
## 10.11 独立编码练习
> 为“平均订单金额”写出指标定义草案和必要表。
## 10.12 测试或评测验证
> 用现有语义测试验证别名和表达式加载。
## 10.13 面试复述题
> 语义层如何降低业务口径幻觉？
## 10.14 掌握度检查与下一章
> 能区分业务指标、物理列和候选。下一章学习 Grounding 与澄清。
