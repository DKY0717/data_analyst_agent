# 第4章 环境、配置与调试方法

> 本章预计 1～2 小时，目标是先学会定位环境问题，再修改业务代码。

## 4.1 学习目标
> 能检查工作目录、依赖、环境变量、端口、日志和异常堆栈。

## 4.2 前置知识
> 需要会使用 PowerShell 或终端执行命令。

## 4.3 为什么需要这一模块
> Vibe Coding 项目常把环境错误误判成代码错误。建立固定排查顺序可以避免无意义修改。

## 4.4 输入、输出与依赖
> Settings 从环境变量产生运行配置；日志和异常输出提供证据；端口与进程决定服务能否启动。

## 4.5 执行流程
```text
确认目录 → 确认解释器和依赖 → 确认配置 → 确认数据库 → 确认端口 → 运行最小测试
```

## 4.6 当前代码地图
| 内容 | 路径 |
|---|---|
| 配置示例 | `.env.example` |
| Settings | `backend/app/config.py` |
| 日志 | `backend/app/utils/logger.py` |
| 异常 | `backend/app/utils/exceptions.py` |

## 4.7 关键代码理解
> 阅读配置时区分默认值、必须值和安全值；阅读异常时关注对外错误码与内部日志是否分离。

## 4.8 最小动手运行
```powershell
Get-Location
python --version
pytest backend/tests/test_health.py -q
```

## 4.9 故障注入实验
> 临时在当前终端设置一个错误模型名并只运行配置导入检查，记录实际读取值后关闭终端恢复；不要修改或提交 `.env`。

## 4.10 调试路径与常见误判
> 401、429、DNS 和端口占用不是同一种错误。先记录状态、阶段和最小复现，再决定是否进入代码。

## 4.11 独立编码练习
> 为一次假想的“后端启动但 readiness 失败”写出五步排查清单。

## 4.12 测试或评测验证
```bash
pytest backend/tests/test_health.py backend/tests/test_config.py -q
```

## 4.13 面试复述题
> 你如何证明一个错误来自外部模型，而不是 SQL 生成或数据库？

## 4.14 掌握度检查与下一章
> 能在不改业务代码的情况下定位目录、配置和端口问题。下一章学习数据库连接与 Schema Loader。
