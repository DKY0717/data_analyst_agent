# 从 Vibe Coding 到独立开发 Data Analyst Agent

> 这是一套完全独立的新课程，只以当前项目代码为事实来源。课程面向会 Python 基础和简单 SQL、但没有系统开发过 FastAPI、Vue、LangGraph 与 NL2SQL Agent 的学习者。
>
> **课程状态：建设中（32/32 章结构已建立，正文分批深化）**
>
> **首版源码基线：** `8dffc1d76c514c7efe1b6e642ea1880a81989109`
>
> **设计日期：** 2026-07-20

## 1. 为什么重新建立课程

> 项目通过 Vibe Coding 快速形成了大量功能，但“代码存在”不等于“项目所有者理解代码”。这套课程不要求背诵文件，而是按照从零重建当前系统的顺序，解释每个模块为什么出现、输入输出是什么、失败后如何处理，以及如何通过测试和评测证明行为。
>
> 旧课程不会作为前置材料。新课程有自己的目录、代码地图、学习清单、故障手册和面试题，并覆盖当前版本中的真实模型分片、checkpoint、严格汇总和 MiMo 超时事故。

## 2. 三层毕业目标

| 层级 | 完成标准 |
|---|---|
| 能讲清楚 | 不看文档画出架构，解释主要请求路径、安全边界和工程取舍 |
| 能独立修改 | 独立定位错误、增加指标、修改工作流，并补测试与评测 |
| 能从零重写 | 不复制生产源码，完成精简版 NL2SQL Agent 和确定性测试 |

## 3. 八周路线

| 周次 | 章节 | 学习成果 |
|---|---|---|
| 第1周 | 第1～4章 | 能运行、导航和调试项目 |
| 第2周 | 第5～8章 | 能解释并重建最小 NL2SQL |
| 第3周 | 第9～13章 | 能阅读和修改完整 Agent 工作流 |
| 第4周 | 第14～18章 | 能解释安全、权限、重试和审计 |
| 第5周 | 第19～22章 | 能演示前后端产品和部署边界 |
| 第6周 | 第23～25章 | 能写测试并理解专项评测 |
| 第7周 | 第26～28章 | 能解释真实模型证据和 MiMo 事故 |
| 第8周 | 第29～32章 | 能独立开发、重写核心链路并答辩 |

## 4. 七部分课程

> 第一部分补齐阅读当前项目所需的 Python、异步、SQL、配置和调试知识；第二部分搭建最小 NL2SQL；第三部分加入结构化意图、语义、Grounding、LangGraph、Repair 和多轮；第四部分建立安全、权限与可靠性；第五部分完成 API、前端和部署；第六部分学习测试、评测和真实模型证据；第七部分通过独立开发和重写完成毕业验收。

> 完整章节导航位于 [_sidebar.md](_sidebar.md)，源码对应关系位于 [CURRENT-CODE-MAP.md](CURRENT-CODE-MAP.md)。

## 5. 每章学习闭环

```text
明确问题与行为预期
  → 阅读测试和接口
  → 定位当前实现
  → 运行最小验证
  → 注入可恢复故障
  → 根据证据调试
  → 完成小改造
  → 补测试或评测
  → 用自己的语言复述
```

> 每天使用 [STUDY-CHECKLIST.md](STUDY-CHECKLIST.md) 记录实际运行结果。命令失败时先查 [TROUBLESHOOTING.md](TROUBLESHOOTING.md)，不要为了让演示通过而关闭 Guard、权限或严格门禁。

## 6. 真实模型和费用边界

> 大多数学习任务不调用真实模型。课程优先使用 pytest、固定数据和 Fake LLM 建立确定性反馈。只有明确标注的扩展练习才需要 API Key、网络和费用预算。
>
> 不要把 API Key、JWT Secret、数据库密码、请求头或完整供应商响应写入课程、截图、日志或 Git。项目仍保留 `QWEN_*` 变量名用于兼容，但 provider 可以是 MiMo、Qwen 或其他 OpenAI-compatible 服务。

## 7. 学习完成的最终证据

> 课程最终要求提交五类成果：架构图和口头讲解、一个新业务指标、一次完整故障定位、一套测试与评测证据、一个独立重写的精简 Agent。只阅读完三十二章不能视为毕业。
