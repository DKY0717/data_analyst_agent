# 第21章 图表、导出与审计界面
> 本章预计 1～2 小时，学习如何把结果变成可验证、可导出、可解释的产品输出。
## 21.1 学习目标
> 能解释图表选择、表格导出和审计面板的数据边界。
## 21.2 前置知识
> 已掌握查询响应结构、Vue 组件和 SQL 结果表。
## 21.3 为什么需要这一模块
> 自然语言答案容易掩盖细节；结果表、SQL、图表和审计证据共同帮助用户验证结论。
## 21.4 输入、输出与依赖
> 输入是列、行、图表建议与审计摘要；输出是 ECharts 配置、结果表、CSV/XML 文件和审计视图。
## 21.5 执行流程
```text
query result → table/chart/audit panels → optional local export
```
## 21.6 当前代码地图
| 内容 | 路径 |
|---|---|
| 图表 | `frontend/src/components/ChartPanel.vue` |
| 结果表 | `frontend/src/components/ResultTable.vue` |
| 审计面板 | `frontend/src/components/AuditPanel.vue` |
| 导出工具 | `frontend/src/utils/spreadsheet.js` |
## 21.7 关键代码理解
> 导出使用已经返回到浏览器的列与行，不应偷偷再次执行 SQL；审计面板展示摘要而不是 API Key、完整 Prompt 或敏感原始数据。
## 21.8 最小动手运行
```bash
cd frontend
npm run build
```
## 21.9 故障注入实验
> 构造包含逗号、引号、换行和公式前缀的单元格，检查导出是否正确转义。
## 21.10 调试路径与常见误判
> 图表为空时先检查列类型与数据长度；下载成功并不代表电子表格内容安全或编码正确。
## 21.11 独立编码练习
> 为导出文件名增加本地时间戳，并保持特殊字符可用。
## 21.12 测试或评测验证
> 运行前端构建并人工检查同一结果在表格、图表和导出文件中的一致性。
## 21.13 面试复述题
> 为什么自然语言答案之外还要保留 SQL、结果表和审计证据？
## 21.14 掌握度检查与下一章
> 能说明三种展示层的职责与安全边界。下一章完成部署和就绪检查。
