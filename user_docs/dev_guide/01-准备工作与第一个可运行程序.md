# 第一章：准备工作与第一个可运行程序

> 本章目标：搭建开发环境，创建一个能响应 HTTP 请求的最小程序。
>
> 完成本章后，你将能在浏览器中访问一个自己写的 API 接口。

## 1.1 我们要做什么

> 我们要从零开始构建一个"数据分析助手"：用户用自然语言提问，系统自动生成 SQL、查询数据库、返回答案。
>
> 这个项目用到两个核心技术：
> - **Python + FastAPI**：写后端接口
> - **Vue 3**：写前端页面
>
> 不用怕，我会一行一行带你写。

## 1.2 你需要什么

> 在开始之前，确保你的电脑上安装了以下软件。如果没有，按指引安装：

### Git

> Git 用来下载代码。即使你是从零开始写，后面也可能需要版本管理。

```bash
# 验证是否已安装
git --version
# 如果提示找不到命令，去 https://git-scm.com/downloads 下载安装
```

### Python

> 后端语言。我们用 Python 3.11。

```bash
python --version
# 应该输出 Python 3.11.x 或更高
# 如果提示"找不到命令"，试试 py --version
# 如果都没有，去 https://www.python.org/downloads/ 下载
# 安装时务必勾选 "Add Python to PATH"
```

### Node.js

> 前端工具链。我们用 Node.js 18+。

```bash
node --version
npm --version
# 如果没有，去 https://nodejs.org/ 下载 LTS 版本
```

### 编辑器

> 推荐 VS Code：https://code.visualstudio.com/
>
> 免费、轻量、Python 和 Vue 插件丰富。

## 1.3 创建项目目录

> 打开终端（Windows 用户打开 PowerShell——不知道怎么打开？看第 0 章 0.1 节），执行：

```bash
# 创建项目目录
mkdir data_analyst_agent
cd data_analyst_agent

# 初始化 Git
git init
```

> 此时你的目录结构应该是：

```text
data_analyst_agent/
└── .git/
```

## 1.4 创建后端项目骨架

> 我们从最简单的 FastAPI 程序开始。

### 创建目录结构

```bash
# 在项目根目录执行
mkdir -p backend/app
mkdir -p backend/tests
```

### 创建依赖文件

> 创建 `backend/requirements.txt`，声明项目需要的 Python 库：

```bash
# Windows 用户用这个命令创建文件
New-Item -Path "backend/requirements.txt" -ItemType File -Force
```

> 用编辑器打开 `backend/requirements.txt`，写入以下内容：

```text
fastapi==0.104.1
uvicorn[standard]==0.24.0
pydantic==2.5.2
python-dotenv==1.0.0
```

> **逐行解释：**
> - `fastapi`：Web 框架，帮你处理 HTTP 请求和响应
> - `uvicorn`：ASGI 服务器，让 FastAPI 能运行起来
> - `pydantic`：数据校验库，FastAPI 用它来验证请求参数
> - `python-dotenv`：读取 `.env` 配置文件

### 安装依赖

```bash
cd backend
pip install -r requirements.txt
# 如果提示"pip 不是命令"，试试: python -m pip install -r requirements.txt
```

> 验证安装成功：

```bash
pip list | findstr fastapi
# 应该输出类似: fastapi    0.104.1
```

### 创建第一个程序

> **怎么创建带内容的文件？**（以后每个 `.py` 文件都是这样创建）
>
> 方法 1（推荐）：用 VS Code
> 1. 打开 VS Code，菜单 → File → Open Folder → 选择 `data_analyst_agent` 文件夹
> 2. 在左侧文件树中右键 → New File → 输入文件名
> 3. 在编辑区粘贴代码
> 4. 按 `Ctrl+S` 保存
>
> 方法 2：用终端
>
> ```bash
> # 创建空文件
> New-Item -Path "backend/app/__init__.py" -ItemType File -Force
> ```
>
> 然后用编辑器打开文件，粘贴代码并保存。
>
> ---

> 创建 `backend/app/__init__.py`（空文件，让 Python 把 app 当作包）：

```bash
New-Item -Path "backend/app/__init__.py" -ItemType File -Force
```

> 创建 `backend/app/main.py`，写入：

```python
from fastapi import FastAPI

app = FastAPI(title="Data Analyst Agent")

@app.get("/health")
async def health_check():
    return {"status": "healthy"}
```

> **逐行解释：**
> - `from fastapi import FastAPI`：导入 FastAPI 框架
> - `app = FastAPI(...)`：创建一个 FastAPI 应用实例
> - `@app.get("/health")`：定义一个 GET 请求的路由，访问 `/health` 时触发
> - `async def health_check()`：异步函数，处理请求并返回结果
> - `return {"status": "healthy"}`：返回一个字典，FastAPI 会自动转为 JSON

### 运行

```bash
# 在项目根目录下运行（不是 backend 目录）
uvicorn app.main:app --app-dir backend --reload --port 8000
```

> `--app-dir backend` 告诉 uvicorn 去 `backend/` 目录找 `app.main` 模块，但工作目录仍然是项目根目录。
>
> 你应该看到类似输出：

```text
INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
INFO:     Started reloader process
INFO:     Started server process
INFO:     Waiting for application startup.
INFO:     Application startup complete.
```

### 验证

> 打开浏览器，访问 http://localhost:8000/health
>
> 你应该看到：

```json
{"status": "healthy"}
```

> 再访问 http://localhost:8000/docs，你会看到 FastAPI 自动生成的 API 文档页面。
>
> **恭喜！你的第一个 API 已经运行起来了。**

## 1.6 常见问题

### `uvicorn` 命令找不到

> 确保你执行了 `pip install -r requirements.txt`，并且在**项目根目录**下运行命令。
>
> 如果还是找不到，试试：

```bash
python -m uvicorn app.main:app --app-dir backend --reload --port 8000
```

### 端口被占用

> 如果 8000 端口被其他程序占用，换一个端口：

```bash
uvicorn app.main:app --app-dir backend --reload --port 8001
```

### `python` 命令找不到

> Windows 用户试试 `py` 代替 `python`。如果还是不行，检查安装时是否勾选了 "Add Python to PATH"。
