# 第零章：开始之前——你需要知道的基础知识

> 本章面向完全没有编程经验的读者。
>
> 如果你已经会用终端、知道什么是 Python，可以跳过本章。

## 0.1 什么是终端

> 终端（Terminal）是一个**文字界面**。你输入命令，电脑执行命令，然后显示结果。
>
> 和平时用鼠标点图标不同，终端用文字操作电脑。

### 怎么打开终端

> **Windows 10/11：**
> 1. 按键盘上的 `Win` 键（键盘左下角有 Windows 图标的键）
> 2. 输入 `powershell`
> 3. 点击 "Windows PowerShell"
>
> 打开后你会看到一个蓝色背景的窗口，光标闪烁等待你输入命令。

### 终端的基本操作

```bash
# 查看当前在哪个目录（文件夹）
pwd

# 进入某个目录
cd 目录名

# 回到上一级目录
cd ..

# 查看当前目录下有哪些文件和文件夹
ls

# 创建文件夹
mkdir 文件夹名
```

> **举例：** 你在 `C:\Users\你的名字` 目录下，想进入桌面：
>
> ```bash
> cd Desktop
> ```
>
> 想回到上一级：
>
> ```bash
> cd ..
> ```

## 0.2 什么是 Python

> Python 是一种编程语言。你写的代码（`.py` 文件）需要 Python 来执行。
>
> 就像 `.docx` 文件需要 Word 打开一样，`.py` 文件需要 Python 执行。

### 怎么运行 Python 代码

> 方法 1：在终端中运行
>
> ```bash
> python 文件名.py
> ```
>
> 方法 2：在 VS Code 中运行
> 1. 打开 VS Code
> 2. 打开 `.py` 文件
> 3. 点击右上角的运行按钮（三角形）

### pip 是什么

> `pip` 是 Python 的包管理器。包（package）就是别人写好的代码库，你可以直接用。
>
> 安装一个包：
>
> ```bash
> pip install 包名
> ```
>
> 安装 `requirements.txt` 里的所有包：
>
> ```bash
> pip install -r requirements.txt
> ```
>
> `requirements.txt` 就是一个清单，列出了项目需要的所有包和版本号。

## 0.3 什么是 HTTP 请求

> HTTP 是浏览器和服务器之间的"对话协议"。
>
> 当你在浏览器输入一个网址并回车，浏览器就发出了一个 HTTP 请求。服务器收到请求后返回网页内容。
>
> 我们的项目就是做一个"服务器"，接收请求并返回数据。

### 请求的类型

> - **GET**：获取数据。比如浏览器访问一个网页就是 GET 请求。
> - **POST**：提交数据。比如用户在表单里填信息并点"提交"。
>
> 我们的查询接口用 POST，因为用户要提交一个问题。

## 0.4 什么是 API

> API（Application Programming Interface）就是程序对外提供的"接口"。
>
> 打个比方：餐厅的菜单就是 API。你（客户端）看菜单（API 文档），告诉服务员（HTTP 请求）你要什么菜（参数），厨房（服务器）做好后端上来（响应）。
>
> 我们的项目提供这些 API：
>
> | 接口 | 方法 | 作用 |
> |------|------|------|
> | `/health` | GET | 检查服务是否正常 |
> | `/api/schema` | GET | 获取数据库表结构 |
> | `/api/chat/query` | POST | 提交数据问题，返回结果 |

## 0.5 什么是 JSON

> JSON 是一种数据格式，长得像这样：
>
> ```json
> {
>   "name": "张三",
>   "age": 25,
>   "skills": ["Python", "SQL"]
> }
> ```
>
> 规则很简单：
> - 用 `{}` 包裹一个对象
> - 用 `key: value` 存数据
> - 用 `[]` 表示列表
> - 文字用双引号 `""` 包裹
>
> API 之间传递数据通常用 JSON 格式。

## 0.6 什么是异步（async/await）

> 你写的代码通常是"同步"的：一行执行完才执行下一行。
>
> ```python
> # 同步：先执行 step1，等它完成后才执行 step2
> step1()
> step2()
> ```
>
> 但有些操作很慢，比如调用网络 API 可能要等 3 秒。如果一直等着，程序就"卡住"了。
>
> "异步"的意思是：发出请求后不傻等，先去做别的事，等响应回来了再处理。
>
> ```python
> # 异步：发出请求后不等待，可以做其他事
> result = await call_api()  # 这行会"暂停"，但不阻塞其他任务
> ```
>
> `async def` 声明这是一个异步函数，`await` 表示"等这个慢操作完成"。
>
> FastAPI 框架自动处理异步，你只需要在函数前加 `async`，在调用慢操作时加 `await`。

## 0.7 什么是装饰器（@xxx）

> 装饰器是一种"给函数添加功能"的写法。
>
> ```python
> @app.get("/health")
> async def health_check():
>     return {"status": "healthy"}
> ```
>
> `@app.get("/health")` 的意思是：当有人访问 `/health` 这个地址时，执行下面的 `health_check` 函数。
>
> 你不需要理解装饰器的原理，只需要知道 `@app.get("/路径")` = "这个函数处理 GET 请求"。

## 0.8 什么是上下文管理器（with 语句）

> ```python
> with open("文件.txt") as f:
>     content = f.read()
> # 离开 with 块后，文件自动关闭
> ```
>
> `with` 语句保证资源（如文件、数据库连接）用完后一定被释放。即使中间出错也会释放。
>
> 这比手动 `open` + `close` 更安全，不会忘记关闭。

## 0.9 什么是包和模块

> - **模块**：一个 `.py` 文件就是一个模块
> - **包**：一个包含 `__init__.py` 的文件夹就是一个包
>
> ```text
> backend/app/
> ├── __init__.py    ← 这个文件让 Python 把 app 当作"包"
> ├── main.py        ← 这是一个"模块"
> └── db/
>     ├── __init__.py
>     └── connection.py
> ```
>
> `from app.db.connection import db_connection` 的意思是：
> 从 `app` 包的 `db` 子包的 `connection` 模块中，导入 `db_connection` 变量。

## 0.10 什么是 .env 文件

> `.env` 是一个纯文本文件，用来存放配置信息，格式是 `KEY=VALUE`：
>
> ```text
> APP_PORT=8000
> QWEN_API_KEY=your_api_key_here
> ```
>
> 为什么不用代码直接写死配置？
> - 不同环境（开发/测试/生产）配置不同
> - 敏感信息（API Key）不应该写在代码里
> - `.env` 文件在 `.gitignore` 中，不会被提交到 Git
>
> `python-dotenv` 库会在程序启动时读取 `.env`，然后你就能用 `os.getenv("KEY")` 获取配置值。
