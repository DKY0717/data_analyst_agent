## 1. 学习目标

## 2. 技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| Vue 3 | 3.4 | 前端框架，组件化开发 |
| Vite | 5.2 | 开发服务器和构建工具 |
| Element Plus | 2.7 | UI 组件库（按钮、表格、卡片等） |
| Pinia | 2.1 | 状态管理（类似 Vuex，但更简洁） |
| ECharts | 5.5 | 数据可视化图表 |
| Axios | 1.6 | HTTP 请求库 |

## 4. 应用入口

> `main.js` 负责注册 Vue 插件：

```javascript
import { createApp } from 'vue'
import { createPinia } from 'pinia'
import ElementPlus from 'element-plus'
import 'element-plus/dist/index.css'
import App from './App.vue'

const app = createApp(App)
app.use(createPinia())   // 注册状态管理
app.use(ElementPlus)      // 注册 UI 组件库
app.mount('#app')
```

## 5. API 调用封装

> `api/agent.js` 封装了与后端的 HTTP 通信：

```javascript
import axios from 'axios'

const api = axios.create({
  baseURL: '/api',  // Vite 会代理到 localhost:8000
  timeout: 30000,
})

export async function queryQuestion(question, sessionId) {
  const response = await api.post('/chat/query', {
    question,
    session_id: sessionId,
  })
  return response.data
}

export async function getSchema() {
  const response = await api.get('/schema')
  return response.data
}
```

> **代理配置：**
>
> 前端运行在 `localhost:3000`，后端在 `localhost:8000`。Vite 的代理配置把 `/api` 请求转发到后端：

```javascript
// vite.config.js
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:8000',
      changeOrigin: true,
    },
  },
}
```

## 6. Pinia 状态管理

> `stores/query.js` 管理整个应用的状态：

```javascript
import { defineStore } from 'pinia'
import { queryQuestion } from '@/api/agent'

export const useQueryStore = defineStore('query', {
  state: () => ({
    // 当前查询状态
    question: '',
    loading: false,
    result: null,
    error: null,

    // 会话管理
    sessionId: null,
    history: [],

    // UI 状态
    activeTab: 'answer',
  }),

  actions: {
    async submitQuestion(question) {
      this.loading = true
      this.error = null
      this.question = question

      try {
        const response = await queryQuestion(question, this.sessionId)
        this.result = response.data
        this.sessionId = response.data.session_id
        this.history.push({
          question,
          answer: response.data.answer,
          timestamp: new Date(),
        })
      } catch (err) {
        this.error = err.message
      } finally {
        this.loading = false
      }
    },
  },
})
```

> **为什么用 Pinia？**
>
> 多个组件需要共享查询状态（结果、加载状态、历史记录）。Pinia 提供了一个集中的状态管理，避免组件之间层层传递 props。

## 7. 核心组件详解

### 7.1 QueryInput（问题输入）

> 用户输入问题的输入框，支持回车提交和示例问题点击。

```vue
<template>
  <div class="query-input">
    <el-input
      v-model="question"
      placeholder="请输入您的数据问题..."
      @keyup.enter="handleSubmit"
    >
      <template #append>
        <el-button @click="handleSubmit" :loading="loading">
          查询
        </el-button>
      </template>
    </el-input>
  </div>
</template>

<script setup>
import { ref } from 'vue'
import { useQueryStore } from '@/stores/query'

const store = useQueryStore()
const question = ref('')

function handleSubmit() {
  if (question.value.trim()) {
    store.submitQuestion(question.value)
  }
}
</script>
```

### 7.4 ResultTable（结果表格）

> 用 Element Plus 的表格组件展示查询结果。

```vue
<template>
  <el-table :data="tableData" border stripe>
    <el-table-column
      v-for="col in columns"
      :key="col"
      :prop="col"
      :label="col"
    />
  </el-table>
</template>

<script setup>
import { computed } from 'vue'
import { useQueryStore } from '@/stores/query'

const store = useQueryStore()

const columns = computed(() =>
  store.result?.columns?.map(col => ({ prop: col, label: col })) || []
)

const tableData = computed(() => {
  if (!store.result?.rows) return []
  return store.result.rows.map(row => {
    const obj = {}
    store.result.columns.forEach((col, i) => {
      obj[col] = row[i]
    })
    return obj
  })
})
</script>
```

### 7.5 ChartPanel（图表）

> 用 ECharts 自动根据查询结果生成图表。

```vue
<template>
  <div ref="chartRef" class="chart-container"></div>
</template>

<script setup>
import { ref, watch, onMounted } from 'vue'
import * as echarts from 'echarts'
import { useQueryStore } from '@/stores/query'

const chartRef = ref(null)
const store = useQueryStore()

function renderChart() {
  const { columns, rows } = store.result
  if (!columns || !rows || rows.length === 0) return

  const chart = echarts.init(chartRef.value)

  // 简单的柱状图：第一列作为 X 轴，第二列作为 Y 轴
  const option = {
    xAxis: { type: 'category', data: rows.map(r => r[0]) },
    yAxis: { type: 'value' },
    series: [{ type: 'bar', data: rows.map(r => r[1]) }],
  }

  chart.setOption(option)
}

watch(() => store.result, renderChart)
onMounted(renderChart)
</script>
```

## 9. 构建与部署

### 9.1 开发模式

```bash
npm run dev
# 启动 Vite 开发服务器，支持热更新
```

### 9.2 生产构建

```bash
npm run build
# 输出到 dist/ 目录
```

### 9.3 代码分割

> Vite 配置了手动代码分割，把大型依赖单独打包：

```javascript
build: {
  rollupOptions: {
    output: {
      manualChunks: {
        'vendor-vue': ['vue', 'pinia'],
        'vendor-echarts': ['echarts'],
        'vendor-element': ['element-plus'],
      },
    },
  },
}
```

> 这样用户首次访问时只需要下载必要的代码，其他 chunk 按需加载。
