# 项目概览与环境搭建

## 1. 这个项目是什么

> Data Analyst Agent 是一个**自然语言驱动的数据分析系统**。
>
> 用户用中文或英文提出数据问题（比如"销售额最高的前5个商品是什么"），系统会自动生成 SQL、执行查询、修复错误，并用自然语言返回答案和图表。
>
> 简单来说：**你问问题，系统帮你查数据库并解释结果。**

## 2. 技术栈总览

> 项目分为后端和前端两部分，下面列出用到的核心技术：

| 层级 | 技术 | 用途 |
|------|------|------|
| 后端框架 | FastAPI | 提供 HTTP API 接口 |
| AI 编排 | LangGraph | 将多个 AI 步骤串成工作流 |
| LLM | Qwen (通义千问) | 理解自然语言、生成 SQL、生成答案 |
| 数据库 | DuckDB | 本地嵌入式分析型数据库 |
| SQL 解析 | SQLGlot | 安全校验 SQL 语句 |
| 前端框架 | Vue 3 | 用户界面 |
| UI 组件 | Element Plus | 按钮、表格、卡片等 UI 组件 |
| 图表 | ECharts | 数据可视化 |
| 状态管理 | Pinia | 前端数据流管理 |
| 构建工具 | Vite | 前端开发服务器和打包 |
| 容器化 | Docker | 一键部署 |

## 3. 系统工作流程

> 当用户提出一个问题时，系统会依次执行以下步骤：

```text
用户提问
  → Intent Guard（安全检查，拦截危险请求）
  → 意图解析（理解用户想查什么）
  → Schema Grounding（把业务概念映射到数据库表）
  → 主动澄清（问题太模糊时追问用户）
  → 加载数据库表结构
  → LLM 生成 SQL
  → SQL Guard（校验 SQL 安全性，自动加 LIMIT）
  → 执行 SQL 查询
  → 如果失败 → LLM 修复 SQL → 重新校验 → 重新执行（最多 3 次）
  → LLM 生成自然语言答案
  → 返回结果给用户
```

## 4. 项目目录结构

> 了解目录结构是阅读代码的第一步。下面是项目根目录的结构说明：

```text

├── backend/                  # 后端 Python 代码

│   │   ├── schema_context/   # 业务语义层（指标、维度定义）
│   │   ├── security/         # 安全防护（Intent Guard + SQL Guard）
│   │   ├── semantic/         # 语义相关工具

│   ├── tests/                # 测试代码
│   ├── evaluation/           # 评测脚本
│   └── requirements.txt      # Python 依赖
├── frontend/                 # 前端 Vue 3 代码
│   ├── src/
│   │   ├── api/              # API 调用封装
│   │   ├── components/       # Vue 组件
│   │   ├── stores/           # Pinia 状态管理
│   │   ├── styles/           # CSS 样式
│   │   └── views/            # 页面视图
│   ├── package.json          # Node.js 依赖
│   └── vite.config.js        # Vite 构建配置
├── database/                 # 数据库初始化脚本
│   ├── init.sql              # 建表 SQL
│   └── seed_data.py          # 填充测试数据
├── docs/                     # 项目文档
├── docker-compose.yml        # Docker 编排
├── .env.example              # 环境变量模板
└── AGENTS.md                 # AI 辅助开发指南
```

## 5. 环境准备

### 5.1 安装 Git

> Git 是版本管理工具，用来下载项目代码。
>
> - 下载地址：https://git-scm.com/downloads
> - Windows 用户下载后一路 Next 即可

验证安装：

```bash
git --version
```

### 5.2 安装 Python

> 后端使用 Python 3.11+。如果你还没有安装 Python，推荐从官网下载：
>
> - 下载地址：https://www.python.org/downloads/
> - 安装时**务必勾选** "Add Python to PATH"，否则命令行找不到 python 命令

验证安装：

```bash
python --version
# 应该输出 Python 3.11.x 或更高版本
```

### 5.3 安装 Node.js

> 前端使用 Node.js 18+。
>
> - 下载地址：https://nodejs.org/
> - 选择 LTS（长期支持）版本即可

验证安装：

```bash
node --version
npm --version
```

### 5.4 获取 Qwen API Key

> 系统使用阿里云的通义千问（Qwen）大模型来理解问题和生成 SQL。你需要一个 API Key：
>
> 1. 注册阿里云账号：https://www.aliyun.com/
> 2. 进入 DashScope 控制台：https://dashscope.console.aliyun.com/
> 3. 创建 API Key 并保存好

## 6. 项目搭建步骤

### 6.1 克隆项目

```bash
git clone <你的项目仓库地址>
cd data_analyst_agent
```

### 6.2 配置环境变量

> 复制环境变量模板，然后填入你的 API Key：

```bash
cp .env.example .env
```

编辑 `.env` 文件，至少填入以下内容：

```text
QWEN_API_KEY=你的API密钥
QWEN_MODEL=qwen-turbo
DATABASE_URL=duckdb:///./data/database.duckdb
APP_PORT=8000
```

### 6.3 启动后端

```bash
# 先在项目根目录生成数据库和测试数据
python -m database.seed_data

# 进入后端目录
cd backend

# 安装 Python 依赖
pip install -r requirements.txt

# 启动后端服务
uvicorn app.main:app --reload --port 8000
```

> 注意：`python -m database.seed_data` 必须在项目根目录运行，因为 `database/` 目录在根目录下。
>
> 启动成功后，打开浏览器访问 http://localhost:8000/docs 可以看到 API 文档。
>
> `--reload` 参数表示代码修改后自动重启，方便开发。

### 6.4 启动前端

```bash
# 新开一个终端，进入前端目录
cd frontend

# 安装 Node.js 依赖
npm install

# 启动开发服务器
npm run dev
```

> 启动成功后，打开浏览器访问 http://localhost:3000 即可看到界面。

### 6.5 验证是否成功

> 在前端界面的输入框中输入一个问题，比如：
>
> - "查询销售额最高的前5个商品"
> - "各地区的客户数量是多少"
>
> 如果能看到 SQL、查询结果和自然语言回答，说明搭建成功。

## 7. Docker 一键启动（可选）

> 如果你安装了 Docker，可以用更简单的方式启动整个项目：

```bash
# 复制环境变量
cp .env.example .env
# 编辑 .env 填入 QWEN_API_KEY

# 启动所有服务
docker-compose up -d
```

> 这会同时启动后端（8000 端口）和前端（3000 端口）。
