## 2. 为什么选 DuckDB

> 项目选择 DuckDB 作为数据库，而不是 MySQL 或 PostgreSQL，原因如下：
>
> - **嵌入式**：不需要安装数据库服务器，Python 直接 `import duckdb` 就能用
> - **分析型**：专为 OLAP（数据分析查询）优化，聚合查询非常快
> - **零配置**：数据库就是一个文件，放在 `data/database.duckdb`
> - **SQL 兼容**：支持标准 SQL，语法接近 PostgreSQL

## 3. 数据模型总览

> 项目模拟了一个**电商系统**的业务数据，包含 8 张表：

```text
regions（地区）
  ↓ 1:N
customers（客户）
  ↓ 1:N
orders（订单）
  ↓ 1:N         ↓ 1:1
order_items    payments（支付）
  ↓ N:1         ↓ 1:1
products（商品） refunds（退款）
  ↓ N:1
categories（类别）
```

> 简单来说：一个地区有多个客户，一个客户有多个订单，每个订单有多个商品明细，每个订单对应一笔支付，退款订单有退款记录。

## 4. 各表结构详解

### 4.1 地区表（regions）

> 存储客户所在地区信息，是客户表的外键引用目标。

```sql
CREATE TABLE IF NOT EXISTS regions (
    region_id INTEGER PRIMARY KEY,    -- 地区ID，主键
    region_name VARCHAR,              -- 大区名称（East, South, North, West, Central）
    province VARCHAR,                 -- 省份
    city VARCHAR                      -- 城市
);
```

> 测试数据包含 10 个地区，覆盖华东、华南、华北、华西、华中五个大区。

### 4.2 客户表（customers）

> 存储客户基本信息，通过 `region_id` 关联地区表。

```sql
CREATE TABLE IF NOT EXISTS customers (
    customer_id INTEGER PRIMARY KEY,  -- 客户ID，主键
    customer_name VARCHAR,            -- 客户姓名
    gender VARCHAR,                   -- 性别（Male/Female）
    age INTEGER,                      -- 年龄
    region_id INTEGER,                -- 所属地区ID，外键
    register_date DATE,               -- 注册日期
    FOREIGN KEY (region_id) REFERENCES regions(region_id)
);
```

> 测试数据生成了 100 个客户，随机分配到 10 个地区。

### 4.3 商品类别表（categories）

> 商品的分类目录。

```sql
CREATE TABLE IF NOT EXISTS categories (
    category_id INTEGER PRIMARY KEY,  -- 类别ID，主键
    category_name VARCHAR             -- 类别名称
);
```

> 包含 8 个类别：Electronics、Clothing、Food、Home、Books、Sports、Beauty、Toys。

### 4.4 商品表（products）

> 存储商品信息，包含价格和成本。

```sql
CREATE TABLE IF NOT EXISTS products (
    product_id INTEGER PRIMARY KEY,   -- 商品ID，主键
    product_name VARCHAR,             -- 商品名称
    category_id INTEGER,              -- 所属类别ID，外键
    price DECIMAL(10,2),              -- 售价
    cost DECIMAL(10,2),               -- 成本价
    created_at DATE,                  -- 上架日期
    FOREIGN KEY (category_id) REFERENCES categories(category_id)
);
```

> 每个类别 5 个商品，共 40 个商品。`price - cost` 就是利润空间。

### 4.5 订单表（orders）

> 存储订单主信息。

```sql
CREATE TABLE IF NOT EXISTS orders (
    order_id INTEGER PRIMARY KEY,     -- 订单ID，主键
    customer_id INTEGER,              -- 下单客户ID，外键
    order_date DATE,                  -- 下单日期
    status VARCHAR,                   -- 订单状态（Completed/Cancelled/Refunded）
    total_amount DECIMAL(10,2),       -- 订单总金额
    FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
);
```

> 订单状态有三种：
> - `Completed`：已完成
> - `Cancelled`：已取消
> - `Refunded`：已退款

### 4.6 订单明细表（order_items）

> 一个订单可以包含多个商品，这张表记录每个商品的购买详情。

```sql
CREATE TABLE IF NOT EXISTS order_items (
    item_id INTEGER PRIMARY KEY,      -- 明细ID，主键
    order_id INTEGER,                 -- 所属订单ID，外键
    product_id INTEGER,               -- 商品ID，外键
    quantity INTEGER,                 -- 购买数量
    unit_price DECIMAL(10,2),         -- 下单时的单价
    FOREIGN KEY (order_id) REFERENCES orders(order_id),
    FOREIGN KEY (product_id) REFERENCES products(product_id)
);
```

> `unit_price` 记录的是下单时的价格，而不是商品表中当前的价格。这是电商系统的常见设计——历史订单的价格应该冻结。

### 4.7 支付表（payments）

> 记录每笔订单的支付信息。

```sql
CREATE TABLE IF NOT EXISTS payments (
    payment_id INTEGER PRIMARY KEY,   -- 支付ID，主键
    order_id INTEGER,                 -- 关联订单ID，外键
    payment_method VARCHAR,           -- 支付方式（Alipay/WeChat Pay/Credit Card/Debit Card）
    payment_status VARCHAR,           -- 支付状态（Paid/Failed/Pending）
    paid_amount DECIMAL(10,2),        -- 实付金额
    paid_at TIMESTAMP,                -- 支付时间
    FOREIGN KEY (order_id) REFERENCES orders(order_id)
);
```

### 4.8 退款表（refunds）

> 只有状态为 `Refunded` 的订单才有退款记录。

```sql
CREATE TABLE IF NOT EXISTS refunds (
    refund_id INTEGER PRIMARY KEY,    -- 退款ID，主键
    order_id INTEGER,                 -- 关联订单ID，外键
    refund_amount DECIMAL(10,2),      -- 退款金额
    refund_reason VARCHAR,            -- 退款原因
    refund_date DATE,                 -- 退款日期
    FOREIGN KEY (order_id) REFERENCES orders(order_id)
);
```

> 退款原因包括：Quality Issue、Not as Described、Wrong Item、Changed Mind、Late Delivery、Damaged in Transit。

## 5. 测试数据生成脚本

> 测试数据由 `database/seed_data.py` 生成。核心设计如下：

### 5.1 固定随机种子

```python
random.seed(42)
```

> 每次运行都生成完全相同的数据，保证测试结果可复现。这在开发和调试中非常重要——你看到的数据和别人看到的完全一样。

### 5.2 数据生成逻辑

> 脚本按以下顺序生成数据：

```text
1. 地区数据（10 条，硬编码）
2. 类别数据（8 条，硬编码）
3. 商品数据（40 条，每个类别 5 个）
4. 客户数据（100 条，随机分配地区）
5. 订单数据（每个客户 1-5 个订单，每个订单 1-4 个商品）
6. 支付数据（每个订单一条支付记录）
7. 退款数据（只有 Refunded 状态的订单才有）
```

### 5.3 运行方式

```bash
cd backend
python -m database.seed_data
```

> 运行后会看到类似输出：

## 6. ER 图（实体关系图）

> 用文字描述各表之间的外键关系：

```text
regions.region_id  ←──  customers.region_id

customers.customer_id  ←──  orders.customer_id

categories.category_id  ←──  products.category_id

orders.order_id  ←──  order_items.order_id
products.product_id  ←──  order_items.product_id

orders.order_id  ←──  payments.order_id

orders.order_id  ←──  refunds.order_id
```
