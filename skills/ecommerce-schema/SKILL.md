# E-Commerce Schema Reference

## Overview

This project uses an e-commerce database with 8 tables. All SQL generation and testing must use this exact schema.

## Tables

| Table | Purpose | Key Fields |
|-------|---------|------------|
| regions | 地区信息 | region_id, region_name, province, city |
| customers | 用户信息 | customer_id, customer_name, gender, age, region_id, register_date |
| categories | 商品类别 | category_id, category_name |
| products | 商品信息 | product_id, product_name, category_id, price, cost, created_at |
| orders | 订单表 | order_id, customer_id, order_date, status, total_amount |
| order_items | 订单明细 | item_id, order_id, product_id, quantity, unit_price |
| payments | 支付记录 | payment_id, order_id, payment_method, payment_status, paid_amount, paid_at |
| refunds | 退款记录 | refund_id, order_id, refund_amount, refund_reason, refund_date |

## Key Relationships

```
regions ←── customers
categories ←── products
customers ←── orders ←── order_items ──→ products
                ├── payments
                └── refunds
```

## Common Query Patterns

- 销售额统计: `orders` + `DATE_TRUNC('month', order_date)`
- 商品分析: `order_items` + `products` + `categories`
- 地区分析: `customers` + `regions` + `orders`
- 退款分析: `refunds` + `orders`
- 支付方式分析: `payments` + `payment_method`

## Schema Loader Output Format

Schema loader reads from `information_schema` and returns:
```json
{
  "tables": {
    "orders": {
      "columns": [
        {"name": "order_id", "type": "INTEGER", "is_pk": true},
        {"name": "customer_id", "type": "INTEGER", "is_fk": true}
      ]
    }
  }
}
```

## Common Mistakes

| Mistake | Fix |
|---------|-----|
| Using `GETDATE()` | DuckDB uses `CURRENT_DATE` or `NOW()` |
| Using `LIMIT` without ORDER BY | Add explicit ORDER BY for deterministic results |
| Assuming PostgreSQL syntax | DuckDB uses slightly different date functions |
| Wrong table relationships | Check foreign keys listed above |
