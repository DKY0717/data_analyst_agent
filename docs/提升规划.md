## 二、高优先级改进（预计提升 8-10 分）

### 2.1 执行结果对比评测（Execution Accuracy）

**现状**：目前 Spider 评测只做 SQL 字符串标准化比较

**目标**：实现 BIRD 风格的执行结果对比

**改进点**：
- 在 `backend/evaluation/spider_evaluator.py` 中增加 `_compare_execution_result` 方法
- 对参考 SQL 和生成 SQL 分别执行，比较返回的结果集是否一致
- 处理列名差异、排序差异、浮点精度差异等边界情况

**实现思路**：
```python
def _compare_execution_result(self, reference_sql: str, generated_sql: str, db_connection) -> bool:
    """比较两条 SQL 的执行结果是否一致"""
    ref_result = db_connection.execute(reference_sql).fetchall()
    gen_result = db_connection.execute(generated_sql).fetchall()
    
    # 标准化结果集（排序、去重、类型归一化）
    ref_normalized = self._normalize_result_set(ref_result)
    gen_normalized = self._normalize_result_set(gen_result)
    
    return ref_normalized == gen_normalized
```

**价值**：SQL 字符串不同但语义正确的情况会被正确计分，评测更准确

**预计工作量**：2-3 天

### 2.2 Schema 向量检索层

**现状**：全量注入 Schema 到 LLM prompt

**目标**：当表数量 > 20 时，动态检索相关表

**改进点**：
- 新增 `backend/app/semantic/schema_retriever.py`
- 对表名、列名、注释做 embedding，存入向量库（可用 FAISS 或 ChromaDB）
- 在 `backend/app/agents/grounding.py` 的 `_ground_metric` 中加入检索逻辑

**实现思路**：
```python
class SchemaRetriever:
    """基于向量检索的 Schema 选择器"""
    
    def __init__(self, embedding_model=None):
        self.embedding_model = embedding_model or SentenceTransformer('all-MiniLM-L6-v2')
        self.index = None
        self.schema_items = []
    
    def build_index(self, schema_metadata: dict):
        """构建 Schema 向量索引"""
        texts = []
        for table_name, table_info in schema_metadata['tables'].items():
            # 表名 + 列名 + 注释拼接为文本
            text = f"{table_name}: {', '.join(table_info['columns'])}"
            texts.append(text)
        
        embeddings = self.embedding_model.encode(texts)
        self.index = faiss.IndexFlatL2(embeddings.shape[1])
        self.index.add(embeddings)
        self.schema_items = list(schema_metadata['tables'].keys())
    
    def retrieve(self, question: str, top_k: int = 5) -> list[str]:
        """根据问题检索相关表"""
        query_embedding = self.embedding_model.encode([question])
        distances, indices = self.index.search(query_embedding, top_k)
        return [self.schema_items[i] for i in indices[0]]
```

**价值**：支持大规模 Schema 场景，减少 token 消耗

**预计工作量**：3-4 天

### 2.3 多轮对话持久化

**现状**：`backend/app/agents/session_store.py` 基于内存字典

**目标**：支持服务重启后恢复会话

**改进点**：
- 新增 `backend/app/db/session_repository.py`
- 用 SQLite 或 Redis 存储会话状态
- SessionStore 改为读写 repository，保留内存缓存加速

**实现思路**：
```python
class SessionRepository:
    """会话持久化存储"""
    
    def __init__(self, db_connection):
        self.db = db_connection
        self._ensure_table()
    
    def _ensure_table(self):
        self.db.execute("""
            CREATE TABLE IF NOT EXISTS sessions (
                session_id TEXT PRIMARY KEY,
                context_summary TEXT,
                pending_clarification TEXT,
                created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
                updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
            )
        """)
    
    def save_session(self, session_id: str, data: dict):
        """保存或更新会话"""
        self.db.execute("""
            INSERT OR REPLACE INTO sessions (session_id, context_summary, pending_clarification, updated_at)
            VALUES (?, ?, ?, CURRENT_TIMESTAMP)
        """, (session_id, data.get('context'), data.get('clarification')))
    
    def load_session(self, session_id: str) -> dict | None:
        """加载会话"""
        result = self.db.execute(
            "SELECT context_summary, pending_clarification FROM sessions WHERE session_id = ?",
            (session_id,)
        ).fetchone()
        if result:
            return {'context': result[0], 'clarification': result[1]}
        return None
```

**价值**：生产可用性提升，支持长时间会话

**预计工作量**：2-3 天

### 3.3 LLM 降级方案

**现状**：LLM 调用失败直接报错

**目标**：提供 fallback 机制

**改进点**：
- 新增 `backend/app/services/llm_fallback.py`
- 主模型失败时尝试备用模型（如 qwen-turbo → qwen-plus）
- 或降级为规则模板（简单查询场景）

**实现思路**：
```python
class LLMFallbackChain:
    """LLM 降级链"""
    
    def __init__(self, primary_client, fallback_clients=None):
        self.primary = primary_client
        self.fallbacks = fallback_clients or []
    
    async def call(self, messages, temperature, stage):
        """依次尝试主模型和降级模型"""
        try:
            return await self.primary.call(messages, temperature, stage)
        except (LLMTimeoutError, LLMError) as e:
            logger.warning(f"主模型失败: {e}")
            for fallback in self.fallbacks:
                try:
                    return await fallback.call(messages, temperature, stage)
                except Exception:
                    continue
            raise  # 所有模型都失败
```

**价值**：系统可用性提升

**预计工作量**：1-2 天
