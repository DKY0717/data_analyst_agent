# v1.5 面试包装文档设计

## 背景

项目已经完成 v1.0 到 v1.4 的权限治理、评测、CI、安全审计导出和前端权限审计面板增强。当前 README 已能说明功能，但 `docs/interview_guide.md` 仍停留在旧测试数量、旧数据权限能力和旧演示路径上。面试时如果只看旧讲述稿，会低估当前项目深度。

v1.5 不继续扩大运行时功能，而是把现有技术深度包装成可直接用于简历、面试和演示的材料。

## 目标

- 让项目从“功能完成”进一步变成“能讲清楚、能展示证据、能回答追问”的简历项目。
- 同步面试稿到当前 v1.4 事实：527 个后端测试、51 个前端单测、17 个 E2E、65 条结构化评测、5 条权限评测。
- 明确讲法重点：LangGraph 编排、三层安全治理、权限策略外部化、行级权限、权限可观测性、CI 质量门禁、安全审计导出、前端审计面板。
- 给出可复制的简历 bullet、30 秒/90 秒介绍、STAR 项目故事、演示清单和证据索引。

## 文档结构

### 1. 更新 `docs/interview_guide.md`

这份文件继续作为“面试前快速复习稿”，内容要更新到最新实现：

- 更新 30 秒介绍里的能力范围。
- 更新简历写法里的测试数和安全治理能力。
- 更新架构链路：Data Permission Guard 包含 YAML 策略、行级过滤和权限观测。
- 更新关键模块表：补充权限策略、权限评测、质量门禁、安全审计导出、前端 AuditPanel。
- 更新高频追问：
  - 怎么证明权限不是只写在 prompt 里。
  - 怎么证明行级过滤被执行。
  - 怎么把安全能力放进 CI。
  - 怎么面试现场展示安全审计报告。
- 更新演示脚本：从 analyst 行级过滤成功、analyst 越权阻断、admin 成功查询三段讲。
- 更新不足与下一步，避免继续提“行级权限”作为未完成项。

### 2. 新增 `docs/resume_project_packet.md`

这份文件作为“简历投递和面试展示包”，比面试稿更短、更结果导向：

- 项目定位：一句话说明这是企业级 NL2SQL / Data Analyst Agent。
- 简历 bullet：3 个版本，分别面向 Agent 开发、后端工程、AI 应用工程。
- 30 秒介绍和 90 秒介绍。
- STAR 故事：
  - SQL 安全闭环。
  - 结果正确性从 5/10 到 10/10。
  - 权限治理从字段阻断到行级过滤和 CI 门禁。
- 演示清单：本地启动、角色切换、问题输入、审计面板观察点、报告导出命令。
- 证据索引：每个亮点对应代码文件、测试文件、评测命令和文档。
- 面试风险边界：哪些能力已完成，哪些属于下一步扩展。

### 3. 更新 `README.md`

README 不塞入长篇面试材料，只增加一个清晰入口：

- 在项目亮点或快速开始附近加入“面试/简历材料”小节。
- 链接 `docs/interview_guide.md` 和 `docs/resume_project_packet.md`。

### 4. 更新 `logs/work-diary.md`

记录 v1.5 设计、计划、实现和验证结果，方便后续接手。

## 非目标

- 不新增后端、前端运行时功能。
- 不改动 CI 配置。
- 不新增真实 LLM 调用。
- 不重写 README 为营销页面。
- 不夸大未完成能力，例如多租户持久化、公开数据集泛化评测、完整 BI 配置器。

## 验证方式

- 链接检查：README 和新文档引用的本地文件存在。
- 事实一致性检查：文档中的测试数、评测数、命令和关键文件与当前仓库一致。
- 未完成标记扫描：新文档不包含未定稿或待补内容。
- `git diff --check` 通过。
