# v1.4 前端权限审计面板增强设计

## 背景

v1.1 后端已经在 `audit_report.permission_observability` 中提供稳定的数据权限摘要，包括是否经过权限检查、是否允许、阻断规则、引用表/字段、行级过滤规则和 SQL 是否被权限层改写。当前前端 `AuditPanel.vue` 只展示身份、LIMIT 注入、阻断规则和 authorization 事件，面试演示时还需要从事件列表里找证据，表达不够直接。

v1.4 目标是把后端已有权限摘要展示成清晰、可扫读、不会泄露策略表达式的前端证据区。

## 用户体验目标

- 面试演示中，用户能一眼看出本次查询是否执行了数据权限检查。
- 对 analyst 行级过滤场景，面板能明确展示：
  - 权限检查已执行。
  - 决策为允许。
  - 触发了 `row_filter_region_scope`。
  - SQL 被权限层改写。
- 对 analyst 越权字段阻断场景，面板能明确展示：
  - 权限检查已执行。
  - 决策为阻断。
  - 阻断规则为 `block_unauthorized_column`。
  - 关联表/字段只展示名称，不展示完整策略表达式。
- 对没有权限摘要的历史响应或异常响应，前端仍保持兼容，不报错。

## 范围

### 本版要做

- 在 `AuditPanel.vue` 中新增“数据权限”证据区。
- 使用 `report.permission_observability` 作为唯一稳定摘要来源。
- 展示权限状态、决策结果、SQL 改写状态、引用表、引用字段、行级过滤规则和阻断规则。
- 更新 `AuditPanel` 组件单测，覆盖允许行级过滤、阻断字段、缺省摘要兼容。
- 更新权限演示 Playwright mock，让 E2E 能验证权限摘要在真实页面出现。
- 同步 README 的 30 秒演示脚本，加入前端权限摘要观察点。

### 本版不做

- 不新增后端字段。
- 不展示完整权限策略表达式。
- 不展示完整改写后的 SQL。
- 不做权限策略编辑后台。
- 不改动 AgentGraph 执行链路。

## 数据契约

前端只依赖以下字段：

```json
{
  "permission_checked": true,
  "allowed": true,
  "blocked_rule": null,
  "referenced_tables": ["orders"],
  "referenced_columns": ["orders.total_amount"],
  "row_filters_applied": [
    { "table": "orders", "rule_id": "row_filter_region_scope" }
  ],
  "authorized_sql_changed": true
}
```

字段缺省时使用安全默认值：

- `permission_checked`: `false`
- `allowed`: `null`
- `blocked_rule`: `null`
- `referenced_tables`: `[]`
- `referenced_columns`: `[]`
- `row_filters_applied`: `[]`
- `authorized_sql_changed`: `false`

## UI 设计

在现有身份摘要和 LIMIT/阻断规则摘要之后增加“数据权限”区，保持审计面板原有的紧凑后台工具风格。

展示结构：

- 第一行：`权限检查`、`决策`、`SQL 改写` 三个摘要项。
- 第二行：引用表、引用字段、行级过滤规则，以 tag 列表展示。
- 如果存在阻断规则，单独用 danger tag 展示 `blocked_rule`。
- 如果没有 `permission_observability` 或 `permission_checked=false`，显示 `未触发权限检查`，并不渲染空 tag 列表。

文案建议：

- 权限检查：`已检查` / `未触发`
- 决策：`允许` / `阻断` / `未决策`
- SQL 改写：`已改写` / `未改写`
- 小标题：`数据权限`

## 测试策略

### 组件单测

- 允许 + 行级过滤：
  - 展示 `已检查`、`允许`、`已改写`。
  - 展示 `orders` 和 `row_filter_region_scope`。
- 阻断 + 字段越权：
  - 展示 `阻断`、`block_unauthorized_column`、`customers`、`customers.customer_name`。
- 缺省摘要：
  - 不传 `permission_observability` 时组件正常渲染。
  - 展示 `未触发权限检查`。

### E2E

- analyst 成功查询销售额时，mock 响应包含行级过滤摘要，页面审计面板展示 `row_filter_region_scope` 和 `已改写`。
- analyst 查询客户姓名被阻断时，页面审计面板展示 `block_unauthorized_column` 和 `customers.customer_name`。
- admin 查询客户姓名成功时，页面审计面板展示权限检查允许，且 SQL 未改写。
