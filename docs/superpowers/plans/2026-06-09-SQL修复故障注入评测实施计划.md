# SQL Repair 故障注入评测 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 构建独立的确定性 SQL Repair 故障注入评测器，使用真实数据库错误和 Qwen Plus 量化修复成功率、安全性与意图保持能力。

**Architecture:** 新增固定 Repair YAML case、`RepairEvaluationRunner` 与独立 `RepairReportWriter`。评测器先验证并执行安全的错误 SQL 获取真实错误，再调用现有 SQL Repair，修复结果重新经过 Guard 与 DuckDB 执行，最后使用固定片段规则检查意图保持。

**Tech Stack:** Python、pytest、YAML、DuckDB、SQLGlot SQL Guard、Qwen Plus、Markdown/JSON
