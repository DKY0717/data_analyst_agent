## 计划自审

- 覆盖范围：设计规格中的八项目标均映射到明确任务、文件和验收证据。
- 顺序：先修用户可见契约，再修 Agent 正确性和安全边界，随后升级证据/部署，最后加入工程门禁，避免质量工具掩盖功能性问题。
- 安全不变量：任何阶段都不允许 Guard/权限拒绝进入 Repair，不允许绕过 SQL Guard 执行。
- 兼容边界：本地开发保持可用；受保护部署显式 fail-closed。
- 外部依赖：真实 LLM smoke 和远端 CI 需要网络/Secret；无法在本地伪造，其他确定性验收必须完整执行。
