# LLM 调用可观测性与成本评测 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为每次 Qwen 调用采集 Token、耗时、尝试次数与可选成本，并将请求级汇总接入 Agent 审计和两套离线评测报告。

**Architecture:** 新增基于 `ContextVar` 的 LLM Observability 模块，由 `QwenAPIClient` 在 API 边界记录真实 usage 与调用指标。AgentGraph 和独立 Repair Evaluator 在各自请求边界创建隔离轨迹，再由 AuditReport、NL2SQL Evaluation 和 Repair Evaluation 消费统一汇总。

**Tech Stack:** Python、ContextVar、httpx、LangGraph、Pydantic、pytest、Markdown/JSON

---

```python
def start_trace() -> None: ...
```
