# v1.7 真实 LLM 评测分片与证据恢复实施计划

> **执行规则：** 每个实现任务先补失败测试，再完成最小实现；focused verification、规格符合性审查和代码质量审查通过后立即独立提交。不得暂存用户已有的工作日记改动。

**目标：** 将真实模型评测从单 job 串行批次改造成最大并发为 2 的确定性矩阵分片，逐 case 原子保存 checkpoint，并由严格汇总器验证完整证据后生成质量门禁报告。

**设计依据：** `docs/superpowers/specs/2026-07-18-v1.7-real-llm-evaluation-sharding-design.md`

---

**新增文件：**

- `backend/evaluation/shard_support.py`
- `backend/tests/test_shard_support.py`

**步骤：**

- [ ] 先新增分片参数非法、round-robin 稳定性、全集恰好覆盖一次的失败测试。

- [ ] 新增原子写入回归测试，模拟替换前失败时旧 JSON 仍可读取。
- [ ] 实现分片参数值对象、case 文件 SHA-256、确定性 case 选择和元数据构造。
- [ ] 实现通用异步顺序执行器；通过回调复用各评测器的 `evaluate_case` 和 summary 计算逻辑。

**验收：** 共享组件不依赖具体 LLM 或 Agent；任意 case 列表都能稳定分片，并在每条完成后留下有效 JSON。

- [ ] 仅在三份正式报告齐全时运行质量门禁；严格安全审计、summary 和最终 artifact 始终执行。
- [ ] 保持 `enforce_thresholds` 输入语义，不降低任何质量阈值。
- [ ] 运行 workflow 文件、质量门禁、安全审计和项目文档一致性测试。
