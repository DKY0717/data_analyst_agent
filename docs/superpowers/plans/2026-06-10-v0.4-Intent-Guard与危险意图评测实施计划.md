# v0.4 Intent Guard 与危险意图评测 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在 SQL 生成前使用确定性高置信规则阻断危险自然语言意图，并用独立危险意图评测证明阻断率、正常请求通过率和误杀率。

**Architecture:** 新增无外部依赖的 `IntentGuard` 安全模块，并将 `check_intent` 设为 LangGraph 唯一入口节点；阻断请求直接结束工作流并输出稳定审计报告，放行请求继续经过现有 SQL Guard 与执行控制。新增独立确定性 Intent Evaluation，现有 NL2SQL Evaluation 扩展阻断阶段指标，基础 CI 运行 Intent Evaluation。

**Tech Stack:** Python 3.11、LangGraph、pytest、PyYAML、SQLGlot、GitHub Actions、Markdown/JSON

- [ ] **Step 5: 实现最小 Intent Guard**

实现固定规则结构：

```python
class IntentGuard:
    def validate(self, question: str) -> dict:
        normalized = self._normalize(question)
        for rule in self._rules:
            if rule.matches(normalized):
                return {
                    "is_safe": False,
                    "rule_id": rule.rule_id,
                    "reason": rule.reason,
                    "category": rule.category,
                }
        return {
            "is_safe": True,
            "rule_id": None,
            "reason": None,
            "category": None,
        }
```

实现要求：

1. 规则使用“危险动作 + 危险目标”组合，不使用单关键词阻断。
2. 安全绕过规则优先级高于其他规则。
3. 复合请求只要包含明确危险命令就阻断。
4. 中文注释解释规则优先级和降低误杀的设计原因。
5. 返回值不包含原始问题和匹配文本。
6. 暴露全局实例 `intent_guard = IntentGuard()`。

```python
async def _check_intent(self, state: AgentState) -> Dict[str, Any]:
    try:
        result = intent_guard.validate(state["question"])
    except Exception as exc:
        logger.error("Intent Guard 执行失败: %s", type(exc).__name__)
        result = {
            "is_safe": False,
            "rule_id": "block_intent_guard_error",
            "reason": "安全检查暂时不可用",
            "category": "guard_error",
        }

    blocked = not result["is_safe"]
    return {
        "intent_is_safe": result["is_safe"],
        "intent_rule_id": result["rule_id"],
        "intent_category": result["category"],
        "intent_error": result["reason"],
        "answer": (
            f"请求已被安全策略阻断：{result['reason']}"
            if blocked
            else state.get("answer")
        ),
        "audit_events": self._append_audit_event(
            state,
            "intent",
            "check_intent",
            "blocked" if blocked else "success",
            result["reason"] or "用户意图通过安全检查",
            rule_id=result["rule_id"],
            details={"category": result["category"]},
        ),
    }
```

异常日志只记录异常类型，不记录完整用户问题或敏感值。
